2. Grundsatz der Datensparsamkeit

PrepMate ist eine App zur Mahlzeitenplanung und zum Körper-Tracking. Die App ist darauf ausgelegt, so wenig personenbezogene Daten wie möglich zu erheben. Alle von dir eingegebenen Daten werden ausschließlich lokal auf deinem Gerät gespeichert und nicht an externe Server übertragen, sofern dies nicht im Folgenden ausdrücklich beschrieben ist.

3. Lokal gespeicherte Daten

Folgende Daten werden nur auf deinem Gerät in einer lokalen Datenbank (SQLite) gespeichert:

  • Rezepte (Name, Zutaten, Zubereitungsschritte, Fotos)
  • Wochenplan (geplante Mahlzeiten je Tag)
  • Einkaufsliste (Zutaten und Mengen)
  • Körpergewicht (Datum + Gewicht)
  • Körpermaße (z. B. Taillen-, Hüft- oder Brustumfang)
  • Body-Tracking-Fotos (Fortschrittsbilder)
  • App-Einstellungen (z. B. Sprache, Maßeinheiten)

Diese Daten verlassen dein Gerät nicht und werden weder an uns noch an Dritte übermittelt. Eine Synchronisation mit der Cloud findet nicht statt.

4. Kamera-Zugriff

PrepMate benötigt Zugriff auf die Kamera deines Geräts für zwei Funktionen:

  • Barcode-Scanner: Zum Scannen von Lebensmittel-Barcodes, um Nährwertdaten abzurufen. Die Kamera wird nur aktiv, wenn du den Scanner manuell öffnest. Es werden keine Kamerabilder gespeichert.
  • Body-Tracking-Fotos: Du kannst optional Fortschrittsbilder aufnehmen, die ausschließlich lokal auf deinem Gerät gespeichert werden.

Der Kamera-Zugriff erfolgt nur auf ausdrückliche Anforderung durch dich (keine Hintergrund- oder Dauernutzung).

5. Externe Dienste – OpenFoodFacts API

Wenn du in PrepMate einen Barcode scannst oder nach einem Lebensmittel suchst, wird eine Anfrage an die OpenFoodFacts API ( world.openfoodfacts.org ) gesendet. Dabei wird folgende Information übertragen:

  • Der gescannte Barcode oder der eingegebene Suchbegriff
  • Die technische IP-Adresse deines Geräts (durch den Aufbau der HTTPS-Verbindung)

OpenFoodFacts ist eine gemeinnützige, offene Datenbank für Lebensmittelprodukte. Die Übertragung erfolgt verschlüsselt über HTTPS. Für die Datenverarbeitung durch OpenFoodFacts gilt deren eigene Datenschutzerklärung: https://world.openfoodfacts.org/privacy .

Die Nutzung dieser Funktion ist optional. Du kannst Rezepte und Einträge auch vollständig manuell erstellen, ohne die Barcode- oder Suchfunktion zu verwenden.

6. Rezept-Sharing per Link

PrepMate ermöglicht es, Rezepte als Link zu teilen. Dabei werden die Rezeptdaten (Name, Zutaten, Zubereitung) Base64-kodiert in den Link eingebettet. Der Link wird direkt mit der von dir gewählten App (z. B. Messenger, E-Mail) geteilt. Es findet keine Übertragung an unsere Server statt. Der Empfänger des Links kann das Rezept in seiner PrepMate-App importieren.

7. Hinweis zu Gesundheitsdaten und BMI

PrepMate verarbeitet Gewichts- und Körpermaßdaten, die als Gesundheitsdaten im Sinne der DSGVO eingestuft werden können. Diese Daten werden ausschließlich lokal gespeichert und nicht weiterverarbeitet oder geteilt.

Wichtiger Hinweis: Der in der App angezeigte Body-Mass-Index (BMI) dient ausschließlich zur allgemeinen Information. Er stellt keine medizinische Diagnose dar und ersetzt keine ärztliche Beratung. Bitte wende dich bei gesundheitlichen Fragen an eine medizinische Fachkraft.

8. App-Berechtigungen (Android)

PrepMate benötigt folgende Android-Berechtigungen:

  • INTERNET: Für den Zugriff auf die OpenFoodFacts API beim Barcode-Scannen und bei der Lebensmittelsuche.
  • CAMERA: Für den Barcode-Scanner und das Aufnehmen von Body-Tracking-Fotos.
  • READ_MEDIA_IMAGES: Zum Auswählen von Fotos aus der Galerie für Rezept- und Body-Tracking-Fotos (Android 13+).
  • WAKE_LOCK: Damit der Bildschirm während des Koch-Modus aktiv bleibt.

Berechtigungen werden nur angefragt, wenn die entsprechende Funktion genutzt wird. Du kannst Berechtigungen jederzeit in den Android-Systemeinstellungen widerrufen.
